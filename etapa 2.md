# Projeto Final — Desenvolvimento Ágil com Scrum · Etapa 2

**Disciplina:** Engenharia de Software · Prof. Gaio Belitardo de Oliveira  
**Projeto:** FLORESCER — Plataforma digital para coletivos femininos de SJdBV

---

# Objetivo da Etapa

Nesta etapa foi realizado o planejamento da **Sprint 1** do projeto FLORESCER, seguindo a metodologia Scrum.

O foco desta Sprint é a construção do **Produto Mínimo Viável (MVP)**, contemplando apenas as funcionalidades essenciais classificadas como **Must Have** durante a priorização realizada na Etapa 1.

Como o projeto possui foco em Engenharia de Software e Modelagem UML, a definição de concluído ("Definition of Done") consiste na elaboração e validação dos diagramas que representam a arquitetura e o comportamento do sistema.

---

# Release Plan

O projeto foi organizado em **3 Sprints**, com duração de **7 dias** cada.

| Sprint | Objetivo |
|---------|----------|
| Sprint 1 | Modelagem do MVP (Must Have) |
| Sprint 2 | Modelagem das funcionalidades Should Have |
| Sprint 3 | Modelagem das funcionalidades Could Have e refinamentos |

---

# Sprint 1 — MVP

## Objetivo

Projetar a estrutura básica do sistema FLORESCER, garantindo que as funcionalidades essenciais estejam corretamente modeladas e integradas.

---

## Histórias de Usuário contempladas

Foram selecionadas todas as histórias classificadas como **Must Have**:

| ID | User Story |
|----|------------|
| US01 | Visualizar lista de coletivos |
| US02 | Visualizar coletivos no mapa |
| US03 | Visualizar perfil do coletivo |
| US04 | Consultar contatos de emergência |
| US06 | Cadastrar coletivo |
| US07 | Atualizar informações do coletivo |
| US09 | Login da liderança |
| US13 | Responsividade para dispositivos móveis |

---

# Entregáveis da Sprint

Conforme definido pelo professor, a Sprint 1 contempla a modelagem dos seguintes artefatos UML:

- Diagrama de Contexto
- Diagrama de Casos de Uso
- Diagrama de Sequência (fluxo principal)
- Diagrama de Classes do domínio

Todos os diagramas encontram-se na pasta `diagramas/`.

---

# Definition of Done (DoD)

A Sprint será considerada concluída quando:

- Todas as histórias Must Have estiverem contempladas pelos modelos;
- Os diagramas UML estiverem consistentes entre si;
- A arquitetura representar corretamente o funcionamento do MVP;
- Toda a documentação estiver versionada no GitHub.

---

# Considerações

A Sprint 1 representa a base arquitetural do projeto FLORESCER. Nesta etapa foram modeladas apenas as funcionalidades indispensáveis ao funcionamento do sistema, permitindo validar a estrutura do produto antes da evolução para as próximas Sprints.

As funcionalidades classificadas como **Should Have** e **Could Have** serão incorporadas nas etapas seguintes, conforme o planejamento definido no Release Plan.
